    - When title says "Initiating Long" but weight_to < weight_from, cross_check overrides to partial_close and logs WARNING
    - When no $TICKER in title/body, spaCy fallback attempted, parse_method set to "spacy"

  <implementation>
Implement in `bravos/ingestion/parser.py`:

1. **`infer_action_from_title(title: str) -> str | None`** — case-insensitive title keyword matching. Order matters: check "booking partial profits" BEFORE "booking profits" (substring overlap). Return None if no keyword matches.

2. **`cross_check_action(title_action: str | None, weight_from: int | None, weight_to: int | None) -> str | None`** — if both title_action and weight direction available, check compatibility. If conflict, log WARNING and use weight direction (more authoritative). Mapping: weight_to > weight_from = open_or_add, weight_to < weight_from and weight_to > 0 = partial_close, weight_to == 0 = close.

3. **`score_confidence(ticker, action_type, weight_from, weight_to) -> str`** — count non-None fields out of 4. 4=high, 3=medium, <3=low. Reference price excluded per D-10.

4. **`parse_signal(title: str, body: str) -> dict`** — main entry point:
   a. Extract tickers from title+body using TICKER_RE. If len(tickers) > 1, force confidence="low", set ticker=None.
   b. Extract reference_price from body using PRICE_RE (first match).
   c. Extract weight_from, weight_to from body using WEIGHT_RE (body only, not title).
   d. Infer action_type from title via infer_action_from_title().
   e. If ticker is None and no $TICKER found, attempt spaCy fallback (lazy load). Set parse_method="spacy". Note: spaCy extracts ORG entities but cannot resolve to ticker symbol — store with confidence penalty.
   f. Cross-check action_type vs weight direction.
   g. Score confidence.
   h. Return dict with all fields + parse_method ("regex" or "spacy").

5. **spaCy lazy loading**: `_nlp = None; def get_nlp()` pattern. Load `en_core_web_sm` only when needed. If import fails, log WARNING and return empty result (graceful degradation, not crash).
  </implementation>
</feature>
