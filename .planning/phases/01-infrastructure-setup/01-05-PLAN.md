  <action>
1. Create `infra/schema.sql` containing the EXACT DDL from RESEARCH.md "Exact DDL - All 5 Tables" section. This includes:
   - `CREATE TABLE signals` with columns: signal_id, post_url (UNIQUE), post_title, raw_html, scraped_at, ticker, action_type, weight_from, weight_to, ref_price, parse_status, parse_confidence, parse_notes, signal_status, signal_error. With CHECK constraints on action_type, parse_status, signal_status.
   - `CREATE TABLE orders` with columns: order_id, signal_id (FK), ibkr_order_id, ibkr_perm_id, ticker, action, order_type, quantity, submitted_at, status, status_updated_at, rejection_reason, risk_approved, risk_notes. With CHECK constraints on action, status.
   - `CREATE TABLE position_lots` with columns: lot_id, signal_id (FK), order_id (FK), ticker, weight_at_open, quantity, entry_price, opened_at, status, remaining_qty, closed_at. With CHECK constraints.
   - `CREATE TABLE executions` with columns: execution_id, order_id (FK), lot_id (FK nullable), ibkr_exec_id (UNIQUE), ibkr_perm_id, ticker, side, quantity, fill_price, fill_time, exchange, commission, commission_currency, realized_pnl, received_at. With CHECK constraint on side.
   - `CREATE TABLE broker_positions_snapshot` with columns: snapshot_id, snapshotted_at, account, ticker, sec_type, quantity, avg_cost, matches_internal, discrepancy_note.
   - `CREATE VIEW v_open_positions` — aggregated open lots.
   - All indexes from RESEARCH.md DDL.

   Copy the DDL exactly from RESEARCH.md lines 370-555. Do not modify column names, types, or constraints.

2. SSH into bravos_vm1 and install PostgreSQL 15 from PGDG repo (per D-08):
   ```bash
   sudo apt-get install -y curl ca-certificates
   sudo install -d /usr/share/postgresql-common/pgdg
   sudo curl -o /usr/share/postgresql-common/pgdg/apt.postgresql.org.asc --fail \
     https://www.postgresql.org/media/keys/ACCC4CF8.asc
   sudo sh -c 'echo "deb [signed-by=/usr/share/postgresql-common/pgdg/apt.postgresql.org.asc] \
     https://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" \
     > /etc/apt/sources.list.d/pgdg.list'
   sudo apt-get update
   sudo apt-get install -y postgresql-15 postgresql-client-15
   sudo systemctl enable postgresql
   sudo systemctl start postgresql
   ```

3. Create database and user (per D-09 — separate from opt-trade-vm4):
   ```bash
   sudo -u postgres psql <<'EOF'
   CREATE USER bravos WITH PASSWORD 'change_me_at_deploy';
   CREATE DATABASE bravos_trading OWNER bravos;
   GRANT ALL PRIVILEGES ON DATABASE bravos_trading TO bravos;
   \c bravos_trading
   GRANT ALL ON SCHEMA public TO bravos;
   EOF
   ```

4. Configure pg_hba.conf for local md5 authentication:
   ```bash
   # Add line for bravos user local access
   sudo sh -c 'echo "local   bravos_trading   bravos   md5" >> /etc/postgresql/15/main/pg_hba.conf'
   sudo systemctl reload postgresql
   ```

5. Apply schema:
   ```bash
   PGPASSWORD=change_me_at_deploy psql -U bravos -d bravos_trading -f infra/schema.sql
   ```

6. Verify:
   ```bash
   PGPASSWORD=change_me_at_deploy psql -U bravos -d bravos_trading -c "\dt"
   # Should list: signals, orders, position_lots, executions, broker_positions_snapshot
   PGPASSWORD=change_me_at_deploy psql -U bravos -d bravos_trading -c "\dv"
   # Should list: v_open_positions
   ```

1. On the VM, activate venv and initialize Alembic:
   ```bash
   source ~/venv/bin/activate
   cd ~/bravos  # or wherever the project is cloned
   alembic init alembic
   ```

2. Edit `alembic.ini`:
   - Set `sqlalchemy.url = postgresql://bravos:change_me_at_deploy@localhost/bravos_trading`
   - Note: In production, this will read from environment variable or GCP Secret Manager

3. Edit `alembic/env.py` to support reading DB URL from environment:
   ```python
   import os
   # At the top of run_migrations_online():
   url = os.environ.get("DATABASE_URL", config.get_main_option("sqlalchemy.url"))
   ```

4. Create initial migration that represents the current schema state:
   ```bash
   alembic revision -m "initial_schema" --rev-id=001
   ```
   Edit the generated migration to mark it as the baseline (since schema was applied via SQL). The upgrade() should be empty (schema already exists), and downgrade() should drop all tables.

5. Stamp the database at the initial migration:
   ```bash
   alembic stamp 001
   ```

6. Verify:
   ```bash
   alembic current  # Should show: 001 (head)
   ```

7. Remove skip decorators from `test_schema_tables_exist` and `test_schema_dedup_constraint` in tests/test_infrastructure.py.
