**Step A: Investigate opt-trade-vm4 (SSH in and capture versions)**

SSH into opt-trade-vm4 and run these commands, then paste the output back:

```bash
# OS version
lsb_release -a

# Python version
python3 --version

# IB Gateway version
ls ~/Jts/ibgateway/ 2>/dev/null || ls ~/Jts/ 2>/dev/null

# IBC version and location
ls /opt/ibc/ 2>/dev/null

# IBC config (redacted)
cat ~/ibc/config.ini 2>/dev/null | grep -v -i password | head -30

# Systemd services
systemctl list-units --type=service | grep -i "ibc\|ibgateway\|xvfb"

# PostgreSQL version
psql --version 2>/dev/null

# Chrome/Chromium version
google-chrome --version 2>/dev/null || chromium-browser --version 2>/dev/null

# ibapi location
pip3 show ibapi 2>/dev/null || find ~/venv -name "ibapi" -type d 2>/dev/null | head -3
```

**Step B: Provision bravos_vm1 (per D-01 through D-04)**

Create the service account first, then the VM:

```bash
# Create service account (per D-17)
gcloud iam service-accounts create bravos-vm-sa \
  --display-name="Bravos VM Service Account"

SA_EMAIL="bravos-vm-sa@$(gcloud config get-value project).iam.gserviceaccount.com"

# Create VM (per D-01: e2-standard-2, D-02: 50GB SSD, D-03: us-east, D-04: Ubuntu LTS)
# Use ubuntu-2204-lts unless opt-trade-vm4 investigation shows 20.04
gcloud compute instances create bravos-vm1 \
  --machine-type=e2-standard-2 \
  --boot-disk-size=50GB \
  --boot-disk-type=pd-ssd \
  --image-family=ubuntu-2204-lts \
  --image-project=ubuntu-os-cloud \
  --zone=us-east1-b \
  --service-account="${SA_EMAIL}" \
  --scopes=https://www.googleapis.com/auth/cloud-platform

# Verify SSH
gcloud compute ssh bravos-vm1 --zone=us-east1-b --command="echo 'SSH OK'"
```

When done, paste:
1. The opt-trade-vm4 version output
2. The `gcloud compute instances describe bravos-vm1` output
3. Confirmation that SSH works
